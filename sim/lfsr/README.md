# `sim/lfsr` 目录说明

这个目录用于对 [`rtl/common/lfsr.sv`](/home/chen/FUN/CISLC-O3/rtl/common/lfsr.sv:1) 做一个最小化的 Verilator 仿真验证，并进一步观察 `lfsr_out[1:0]` 这 2 个低位在长期运行时的分布是否趋向均匀。

## 1. 模块功能检验

这里的 C++ testbench 在 [`main.cpp`](/home/chen/FUN/CISLC-O3/sim/lfsr/main.cpp:91) 中实现，验证流程分成两部分：

1. 先对 LFSR 做复位，并检查复位后输出是否为初始种子 `16'hACE1`。
2. 在 `enable=0` 的情况下连续运行 10 个周期，检查输出是否始终保持不变。
3. 在 `enable=1` 的情况下继续运行指定次数，每次更新后都按 RTL 中相同的反馈公式计算期望值，并逐周期做断言，确认模块工作正常。

因此，这个仿真程序首先是在验证 LFSR 模块本身的时序和状态更新逻辑是否正确，而不只是做统计。

## 2. 偏差量定义

在确认模块工作正常之后，程序会统计每次更新后 `lfsr_out[1:0]` 的结果。因为最低两位只有 4 种可能：

- `00`
- `01`
- `10`
- `11`

如果它们是完全均匀分布，那么每一类的理想频率都应该是 `0.25`。

这里定义了一个偏差量：

```text
variance_like =
[(p00 - 0.25)^2 + (p01 - 0.25)^2 + (p10 - 0.25)^2 + (p11 - 0.25)^2] / 4
```

其中 `p00/p01/p10/p11` 是四种结果各自出现的频率。

这个量的含义是：

- 越接近 `0`，说明四种结果越接近均匀分布。
- 越大，说明分布越不均匀。

为了让图像更容易观察，绘图时实际使用的是：

```text
variance_like * 10000
```

## 3. 绘图脚本

[`plot_variance.py`](/home/chen/FUN/CISLC-O3/sim/lfsr/plot_variance.py:1) 用于批量运行仿真并生成曲线图。

它的流程是：

1. 横轴使用总更新次数 `total_updates`。
2. 从较小样本 `100` 开始，到 `65535` 结束。
3. 在线性区间内取多个样本点。
4. 对每个样本点运行一次仿真，提取 `variance_like`。
5. 将 `variance_like * 10000` 作为纵轴，绘制图像 [`variance_curve.png`](/home/chen/FUN/CISLC-O3/sim/lfsr/variance_curve.png)。

这样就可以直观看到：随着更新次数增加，这个偏差量是否会逐步减小，也就是最低两位的分布是否会逐渐接近均匀。

## 4. 图中的参考横线

为了方便比较，图上加了两条参考横线：

1. 理想均匀分布基线：`0`
2. 极端不均匀情况基线：只有 1 类结果占 `100%`，其余 3 类都为 `0%`

对于第 2 种极端情况：

```text
variance_like =
[(1 - 0.25)^2 + 3 * (0 - 0.25)^2] / 4
= 0.1875
```

绘图时乘以 `10000` 后，对应的参考值是：

```text
0.1875 * 10000 = 1875
```

所以图上的红色横线表示“单一结果独占全部样本”的极端不均匀上界，绿色横线表示“理想完全均匀”的下界。

## 5. 使用方法

构建并运行一次单点仿真：

```bash
env CCACHE_DISABLE=1 make -C sim/lfsr run RUN_ARGS=1000
```

生成整张统计曲线图：

```bash
env CCACHE_DISABLE=1 python3 sim/lfsr/plot_variance.py
```

生成的图片默认保存在：

```text
sim/lfsr/variance_curve.png
```
